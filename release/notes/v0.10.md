# Gethes v0.10

## Hotfix
- Fixed startup crash: `ModuleNotFoundError: No module named 'gethes.application'`.
- Included the new `gethes.application` and `gethes.domain` modules in packaged builds.

## Build Pipeline Hardening
- Updated PyInstaller arguments to explicitly collect:
  - `gethes.application` submodules
  - `gethes.domain` submodules
- Added explicit hidden imports for the new architecture modules to avoid missing-package regressions.

## Result
- Installer and portable builds now include the architecture runtime modules required by `gethes.app`.

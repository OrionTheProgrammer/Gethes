# Gethes v0.11

## Syster AI Hotfix
- Fixed the behavior where Syster replied with local scripted answers when the real AI core was not ready.
- Unknown/free-form prompts now report clear Syster Core readiness state instead of fake fallback chatter.
- Unknown terminal commands are no longer auto-routed to Syster unless the AI core is actually ready.

## Runtime Reliability
- Added automatic Windows runtime bootstrap for Syster Core:
  - If `ollama.exe` is not found, Gethes now attempts to download and prepare the Ollama runtime in the user storage directory.
  - Status now surfaces runtime bootstrap progress/errors in `syster core status`.
- Improved Syster Core status output with explicit reason codes for faster diagnostics.

## Internal
- Added test coverage for the new "core unavailable" reply path.

# Documentation Language Policy

All project-facing documentation must be written in English, including:

- `README.md`
- files under `docs/`
- release notes under `release/notes/`
- release descriptions prepared for GitHub Releases

## Scope

This policy applies to:

- architectural documents,
- setup/deployment guides,
- changelogs/release notes,
- contributor-facing operational docs.

## Exceptions

In-game localized content (UI text, narrative, commands shown to players) is not constrained by this policy and remains multilingual by design.
